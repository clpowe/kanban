# Rewards Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add a shared rewards catalog that parents can manage and children can redeem with points.

**Architecture:** Reuse the current server-rendered HTMX layout by adding a reward service layer, reward routes, and a page section for catalog display and redemption actions. Keep redemption state minimal by deducting points directly from `users.points` without introducing a redemption ledger.

**Tech Stack:** Bun, Hono JSX, HTMX, Drizzle ORM, SQLite, Bun test
